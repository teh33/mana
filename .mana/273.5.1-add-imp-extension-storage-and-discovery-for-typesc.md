Goal: add the storage/discovery layer for TypeScript extensions without executing them yet.

Updated storage decision:
- Use project-local `.imp/extensions/`, not `~/.config/imp`.
- Imported Pi extensions should go under `.imp/extensions/pi/<name>/...`.
- Use language-neutral `extensions` naming rather than `ts`/`typescript` so future runtimes fit the same layout.

Steps:
1. Add core detection structs for TypeScript extensions with fields: name, source, source_path, entrypoints, package metadata if present, and detected shape.
2. Scan `.imp/extensions` for:
   - source-namespaced imported packages such as `.imp/extensions/pi/<name>`
   - directory packages with `package.json` and `pi.extensions`
   - fallback directory packages with `index.ts`
   - simple single-file `.ts` extensions if supported under the extension root
3. Keep this separate from Lua discovery so current Lua behavior remains unchanged.
4. Add tests with temp dirs covering package-json, fallback index.ts, single-file detection, and source namespace detection.

Out of scope:
- Executing TypeScript.
- Installing npm dependencies.
- Replacing Lua discovery.

Acceptance:
- Tests prove TypeScript extension detection for `.imp/extensions` and no regression to existing import tests.

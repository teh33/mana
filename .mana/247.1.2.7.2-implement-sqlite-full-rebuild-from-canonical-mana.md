Goal: Implement full SQLite index rebuild from canonical human-editable mana Markdown+YAML files.

Current state:
- 247.1.2.7.1 provides schema and freshness metadata.
- Canonical files remain `.mana/*.md` parsed via Unit/YAML frontmatter.
- Invalid typed files must be recorded and must not be silently indexed as valid.

Steps:
1. Reuse existing Unit parsing (`Unit::from_file` / equivalent) rather than writing a second parser.
2. Implement a full rebuild function that scans the mana root/archive as appropriate, parses canonical files, validates typed fields, and writes SQLite rows through the stable index API.
3. Record invalid/skipped files in SQLite metadata/diagnostics with exact path and parse/schema error; do not let partial stale rows remain trusted for invalid files.
4. Preserve human Markdown bodies/prose; do not rewrite canonical files during rebuild.
5. Add tests with valid units, invalid YAML/frontmatter, stale source metadata, and rebuild idempotence.

Files:
- `mana/crates/mana-core/src/**` (modify — index rebuild module)
- existing unit/index tests (modify/add targeted tests)

In scope:
- Full rebuild only.

Out of scope:
- Incremental write-path updates.
- Context assembly queries.
- Doctor CLI command.

Do not:
- Make SQLite source of truth.
- Auto-fix human files during rebuild.
- Ignore invalid files.

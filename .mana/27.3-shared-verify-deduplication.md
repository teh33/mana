When multiple units share the same verify command (e.g. cargo build, cargo test), run it once and attribute the result to all relevant units.

Current batch_verify is a boolean that defers verify. This replaces it with smarter grouping:
1. After agents complete, collect all pending verify commands
2. Group by exact command string
3. Run each unique command once
4. Attribute pass/fail back to the units that share that command

This directly addresses the pain point of N units all running 'cargo build' separately.

Add VerifyGroup { command: String, unit_ids: Vec<String> } and a group_verify_commands function.

Emit PoolEvent::VerifyGroupRun { command, unit_ids, success } for each group.

Tests: dedup_verify_groups_identical_commands, dedup_verify_runs_once_per_group, dedup_verify_attributes_failure_to_all.

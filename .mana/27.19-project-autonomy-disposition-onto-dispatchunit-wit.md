Goal: prepare the `mana-pool` boundary so any future keep-going work can consume only an explicit derived autonomy disposition projection.

Current state:
- `mana/crates/mana-pool/src/types.rs` `DispatchUnit` includes deterministic scheduling inputs and retry context only.
- `mana/crates/mana-pool/src/dispatch.rs` does not know review state, verify posture, visibility state, or attempt pressure.
- 27.10 concluded that the pool must only consume an explicit projection of the derived disposition and must not invent confidence judgments.

Task:
Define and implement the reduced dispatch-surface projection of the autonomy disposition, without changing dispatch/keep-going behavior yet.

Concrete steps:
1. Inspect `mana/crates/mana-pool/src/types.rs` and `mana/crates/mana-pool/src/dispatch.rs`.
2. Add a reduced autonomy projection to `DispatchUnit` containing only the fields the pool may safely consume later (for example disposition kind, blocker codes, review state, verify posture, visibility state, attempt pressure, risk band, continuation budget).
3. Ensure the pool treats it as data-only input; do not add new keep-going logic in this unit.
4. Add tests or type coverage proving the dispatch surface contains the explicit projection.
5. Document in code/comments that the pool must not inspect raw confidence, model output, or free-text notes.

Files:
- `mana/crates/mana-pool/src/types.rs` (modify)
- optionally nearby dispatch tests if needed for type/constructor coverage (modify)

Do not:
- Do not implement pool-side keep-going behavior here.
- Do not recompute policy in mana-pool.
- Do not pass raw confidence or imp heuristic scores into the pool.

Dependencies / context:
- Depends conceptually on the upstream canonical disposition work from 27.13, 27.14, 27.17, and 27.18.
- This is the last boundary-prep slice before any future keep-going implementation could even be considered.

When dispatching parallel agents in the same repo, each one gets an isolated git worktree. Merge results back on success.

mana-core already has worktree.rs with worktree creation and merge logic. mana-pool should use it instead of relying on file conflict tracking as a workaround.

Add to PoolConfig:
- use_worktrees: bool (default false, opt-in)
- worktree_base: Option<PathBuf> (where to create worktrees, defaults to .mana/worktrees/)

The Spawner receives a worktree path instead of the main repo path when worktrees are enabled. On success, pool calls mana-core worktree merge. On failure, worktree is cleaned up.

The existing file conflict tracking (running_paths) becomes a fallback for when worktrees are disabled.

Tests: worktree_spawner_receives_worktree_path, worktree_cleanup_on_failure, worktree_fallback_when_disabled.

Files: mana/crates/mana-pool/src/types.rs, mana/crates/mana-pool/src/dispatch.rs

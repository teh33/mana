Goal: Add the foundational SQLite derived-index schema and freshness metadata for mana.

Current state:
- Canonical mana units live in human-editable Markdown+YAML files.
- SQLite is planned as a derived/rebuildable index, not source of truth.
- Architecture docs from 247.1.2.5 and validation docs from 247.1.2.6 must be completed first.

Steps:
1. Read the completed docs from 247.1.2.5 and 247.1.2.6 before implementing.
2. Add a mana-core module for the derived SQLite index schema, likely under `mana/crates/mana-core/src/index` or equivalent after inspecting existing index code.
3. Define metadata tables for schema version, index build version, source root, file path, file hash/mtime, last indexed timestamp, and stale/invalid status.
4. Define full typed-schema tables according to the architecture doc. Include stable Rust API functions; do not expose raw SQL/table assumptions to agents.
5. Add tests for schema initialization, schema version checks, recording source metadata, and detecting stale source files.

Files:
- `docs/architecture/mana-sqlite-derived-index.md` (read)
- `docs/architecture/mana-human-edit-validation.md` (read)
- `mana/crates/mana-core/src/**` (modify — exact module after inspection)
- `mana/crates/mana-core/Cargo.toml` (modify only if SQLite dependency is not already available; ask if dependency choice is consequential)

In scope:
- Schema/freshness foundation only.

Out of scope:
- Full index rebuild scanning.
- Write-path integration.
- Context assembly queries.
- Making SQLite canonical.

Do not:
- Change `.mana/*.md` canonical format.
- Let raw SQLite tables become the public API.
- Add daemon/watchers.

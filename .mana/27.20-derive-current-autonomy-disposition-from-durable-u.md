Goal: implement the mana-core policy evaluation step that turns scattered durable facts into the single scheduler-facing autonomy answer defined by 27.10.

Current state:
- 27.10 specified the contract but current code only stores source facts and close/verify outcomes across `mana-core`; there is no canonical evaluator yet.
- `mana/crates/mana-core/src/unit/mod.rs` already stores source fields like `decisions`, `feature`, `verify`, `verify_fast`, `attempts`, `max_attempts`, `history`, and `attempt_log`.
- `mana/crates/mana-core/src/ops/close.rs` already exposes verify/close outcomes like `DeferredVerify`, `VerifyFrozenViolation`, close evidence, and failure/circuit-breaker information.
- Future schedulers must not derive continuation policy for themselves from raw history.

Task:
Add one canonical evaluator in mana-core that derives the current `AutonomyDisposition` (or final equivalent name) from durable unit facts + attempt observations + verify/review/approval state.

Concrete steps:
1. Inspect the vocabulary added by 27.13 and the attempt observation shape from 27.14.
2. Inspect `mana/crates/mana-core/src/unit/mod.rs`, `mana/crates/mana-core/src/unit/types.rs`, `mana/crates/mana-core/src/ops/close.rs`, and `mana/crates/mana-core/src/ops/run.rs` for the durable inputs already available.
3. Implement a canonical evaluation function in mana-core that:
   - emits `blocked` / `requires_human` / `eligible` / `unknown` or equivalent
   - produces typed blocker codes rather than prose
   - maps unresolved decisions to the canonical blocker
   - maps `feature` / human-close requirements to the canonical blocker
   - maps verify state (`failed`, `deferred`, `absent`, frozen-violation, unknown-strength) to typed posture/blockers
   - maps attempt history / retry budget / circuit-breaker state into normalized attempt pressure
   - incorporates review/approval state when those durable inputs exist
   - defaults unknown gate state to non-eligible
4. Store or expose the derived result in the canonical place chosen by 27.13 without making callers replay history.
5. Add focused tests for the evaluator over representative combinations: unresolved decisions, awaiting verify, exhausted attempts, missing visibility, human-close-required, and unknown review/verify state.

Files:
- `mana/crates/mana-core/src/unit/mod.rs` (modify if the derived field lives here)
- `mana/crates/mana-core/src/unit/types.rs` (modify)
- `mana/crates/mana-core/src/ops/close.rs` (read and integrate as needed)
- `mana/crates/mana-core/src/ops/run.rs` (read / integrate only if needed)
- nearby tests in `mana-core` (modify)

Dependencies / context:
- This unit should follow the schema/vocabulary work in 27.13.
- It should consume the attempt-scoped observation shape from 27.14 rather than inventing new ad hoc fields.
- Align any review/approval gate handling with the durable approval/review record work in 45.6.3 rather than inventing a parallel representation.

Do not:
- Do not add or consume raw confidence scores.
- Do not let the evaluator parse free-text notes or verify command text as a proxy for quality.
- Do not change mana-pool dispatch behavior yet.
- Do not duplicate imp-local continuation heuristics in mana-core.

This unit is the bridge between the contract spec and any future scheduler consumption: one upstream evaluator, one explicit typed answer.

## Task
Write a root sequencing doc for introducing first-class durable memory object types into mana without breaking current unit workflows.

## Recommended scope
The document should propose and justify an order such as:
- first semantic checkpoints as explicit handoff objects
- then evidence objects and lightweight evidence refs
- then claims derived from evidence
- then lessons/promoted failure abstractions
- then receipts/env/runbook normalization if they are not already covered elsewhere

## Must explain
- why each object type lands in that order
- what current unit fields it initially coexists with or mirrors
- what later retrieval/view work each new object unlocks
- which object types are optional in the first proving slice versus required for real value

## Files
- `docs/architecture/memory-object-introduction-sequence.md`

## Do not
- do not require all target object types to land at once
- do not pretend claims/evidence can be introduced safely without a compatibility story

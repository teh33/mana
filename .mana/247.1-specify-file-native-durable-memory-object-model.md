## Task
Create a root architecture spec at `docs/architecture/file-native-memory.md` that captures the canonical file-native memory model for autonomous software agents.

## Current design direction
The active direction from discussion is:
- 3-layer architecture: immutable durable objects, rebuildable derived views/indexes, and ephemeral runtime context
- durable truth is modeled as scoped claims backed by evidence and provenance rather than mutable fact records
- accepted/current facts, open questions, stale/conflict summaries, and bootstrap packets are materialized views rather than canonical truth
- durable objects should distinguish observation records from judgment records

## Required sections
1. `## Layer model`
   - define the three layers and what correctness may depend on
   - state that deleting indexes/views must not destroy truth
2. `## First-class durable objects`
   - enumerate `attempt`, `checkpoint`, `evidence`, `claim`, `decision`, `lesson`, `runbook`, `env`, `receipt`
   - optionally mention `question` as an unresolved-state object if useful
   - split the list into observation objects vs judgment objects
3. `## Derived views and indexes`
   - identify accepted facts, project/bootstrap packets, open questions, conflict maps, stale queues, and search indexes as rebuildable views
4. `## Ephemeral runtime context`
   - define scratchpads, prompt packs, caches, and transient tool chatter as non-authoritative

## Constraints to address
- human-readable and git-friendly
- no daemon or database dependency
- supports cold-start inheritance for new agents
- avoids prompt bloat
- works with parallel agents and retries

## Important invariants
- immutable objects are canonical
- current state is materialized, never canonical
- retries add history rather than rewriting failed attempts
- provenance and scope are mandatory, not optional

## Files
- `docs/architecture/file-native-memory.md` (create/modify)

## Do not
- do not describe mutable summary files as source of truth
- do not collapse evidence and claims into a single object type
- do not leave scope/provenance implicit

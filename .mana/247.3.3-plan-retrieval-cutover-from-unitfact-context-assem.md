## Task
Write a root retrieval migration plan that moves mana/imp context assembly from today's unit/fact-oriented retrieval toward object-backed/materialized views.

## Current grounded state
`mana/crates/mana-cli/src/commands/memory_context.rs` currently assembles warnings, working_on, relevant_facts, and recent_work directly from unit files and fact units.

## What to plan
1. The first object-backed views worth introducing for real value
   - bootstrap packet
   - latest checkpoint summary
   - accepted/current claims view or earlier precursor if full claims are not ready
2. How retrieval falls back when those views are absent
3. Which imp-facing context assembly path should consume the new views first
4. How prompt-budget control improves during cutover
5. How to avoid duplicated or contradictory retrieval between unit-native and object-native memory during migration

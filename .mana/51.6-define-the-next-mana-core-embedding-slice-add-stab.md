Current inspected repo reality is captured by closed fact 63: `mana/crates/mana-core/src/api/mod.rs` already exposes broad discovery/query/mutation/orchestration helpers, while several operations remain only under lower-level `mana_core::ops::*` modules (`init`, `sync`, `batch_verify`, `config_get`/`config_set`, `adopt`, `unarchive`, plus other maintenance helpers) and there is no single stable full-spawn embedding entry point. This child exists so the next implementation/planning delta is durable in root mana rather than implied only by the fact.

## Goal
Choose and externalize the smallest coherent next embedding slice for cross-project mana↔imp work:
1. either add a clearly scoped set of top-level `mana_core::api::*` wrappers for the remaining embedding-relevant ops, or
2. explicitly document/decide that those stay lower-level for now and instead define the one stable full-spawn/library entry point boundary.

## Required inspection
- `mana/crates/mana-core/src/api/mod.rs`
- `mana/crates/mana-core/src/ops/mod.rs`
- current run/spawn ownership seams in `mana-cli`, `mana-pool`, and/or `imp` as needed to avoid inventing a fake embedding boundary
- `mana/ARCHITECTURE.md` and any nearby Tower contract docs that already describe intended ownership

## Deliverable
Create or update one durable design artifact or mana note that makes the next step explicit enough for a worker to implement cold. The output must answer:
- which missing wrappers are actually embedding-critical now
- which should remain CLI/maintenance-only for now
- whether the first stable full-spawn/library entry point belongs in `mana-core`, `mana-pool`, `mana-cli`, or `imp`
- what verify/check surface should prove that slice when implemented

## Constraints
- Root scope: this changes the mana↔imp boundary and embedding story
- Do not implement broad cleanup or speculative new runtime surfaces
- Ground every claim in inspected repo files
- Prefer a small reversible next slice over a grand redesign

## Acceptance
A durable root-scope unit or artifact exists that makes the next embedding-surface decision explicit and reviewable, with file ownership, intended scope, and a concrete future verify direction.

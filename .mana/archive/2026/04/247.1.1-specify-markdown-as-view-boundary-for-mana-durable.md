Goal: Add a focused architecture section/spec that makes the Markdown boundary explicit for mana's file-native memory/work substrate.

Current state: Unit 247.1 already defines the broader file-native durable memory object model. The 2026-04-27 Theo/T3 video discussion adds a concrete design pressure: Markdown is useful as a low-friction human authoring/rendering syntax, but becomes unsafe and ambiguous when treated as the canonical programmable substrate.

Steps:
1. Update or create architecture text under `docs/architecture/file-native-memory.md` or a linked small doc to state the boundary: Markdown is a view/envelope/authoring affordance; typed objects are canonical.
2. Define the minimum canonical object families relevant to this boundary: Task, Decision, Blocker, Claim/Assertion, Evidence, Attempt, Artifact, Review, Rule, Event/Receipt.
3. State parser/runtime invariants: agents and schedulers must not infer blocking state, evidence validity, close eligibility, or executable behavior from prose alone; these come from typed fields/objects.
4. State extension/build-system implications: no inline executable semantics hidden in raw Markdown; use explicit hooks/capabilities, schema validation, provenance, and deterministic renderers.
5. Tie the boundary back to current mana reality: current Markdown unit files may remain git-friendly storage/human views during migration, but the target direction is object-native truth with rebuildable Markdown/materialized views.

Files:
- `docs/architecture/file-native-memory.md` (modify — add Markdown-as-view boundary if this doc is the right home)
- optionally `docs/architecture/markdown-boundary-for-mana.md` (create — only if the section would make the main file too noisy)

In scope:
- Architecture/spec language only.
- Clear durable invariants future workers can implement against.

Out of scope:
- Parser implementation.
- Data migration implementation.
- Changing current `.mana/*.md` storage immediately.

Do not:
- Treat Markdown as the canonical source of truth for semantic state.
- Add a new Markdown dialect or embedded execution language.
- Collapse human readability with machine authority.
